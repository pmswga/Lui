Предметной областью курсового проекта являются графические интерфейсы и их создание с помощью GUI библиотек.
Основными задачами, при их создании выделяют: 
1) изучение библиотеки;
2) реализацию графического интерфейса.
Решение первой задачи связанно напрямую со сложностью и документированностью библиотеки.
Решение второй задачи связано с навыками разработчика и выбранным им инструментальным средством: языком или библиотекой.
Подход в создании графического интерфейса разделяется на императивный и декларативный. 
Императивный подход, подразумевает реализацию графического интерфейса путём создания компонентов, как на самом языке, используя библиотеку, так и с помощью визуального редактора, называемого дизайнером форм.
Декларативный подход, включает в себя описание графического интерфейса в виде формата данных (по которому в дальнейшем создаётся интерфейс) или кода на декларативном языке.
Современные библиотеки могут сочетать в себе оба подхода, что позволяет выбирать разработчику тот или иной подход. Как правило, выбор библиотеки зависит от сложности создаваемого интерфейса. Сложные и объёмные библиотеки охватывают задачу создания сложных интерфейсов, в которых может быть много форм, различных и специфичных компонентов.
Однако в рамках задач, требующих от создаваемого интерфейса наличия нескольких кнопок и компонентов для вывода данных, выбор таких библиотеки не является практичным. Такие задачи возникают в научной сфере и можно привести следующий пример, связанный с метеорологическими исследованиями, которые могут проводится в полевых условиях. Полученные данные из программы (написанной на Python), необходимо оценить, чтобы получить итоговый результат исследования. 
В таком случае можно использовать декларативный язык, который нацелен на решение именного такого класса задач, не требующих создания сложного графического интерфейса. Также при создании такого интерфейса могут ещё предъявляться критерии скорости и простоты реализации такого интерфейса.
Исходя из темы курсовой работы таким инструментальным средством должен стать создаваемый декларативный язык, который должен решать задачу создания графического интерфейса для скриптовых программ, написанных на языке Python.
Соответственно целью курсового проекта является создание декларативного языка, автоматизирующего создание графического интерфейса за счёт генерации кода под GUI библиотеку.
Языком, выбранным для реализации транслятора языка является Python, по требованию технического задания. Целевой GUI библиотекой, выбрана Tkinter, так как она входит в стандартный пакет языка и имеет простое API для реализации графического интерфейса.
Основными структурными элементами курсового проекта являются введение, описывающее постановку задачи, основная часть, описывающая вопросы и задачи связанные с созданием языка и заключение, описывающее результаты полученные по итогу выполнения курсового проекта.
